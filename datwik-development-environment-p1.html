Build your own mobile computer (Part 1)

Figuring out development hardware for datwik.com is as much fun as development itself. Goal is to make it possible to use few availed hours in the week here and there regardless of location and be somewhat economical (spoiler alert - mostly failed on second one here).

Microsoft Windows is my base platform to do all the final touches and deployment. Windows 10 with bundled WSL makes sharing the scripts and codebase with all other Linux flavors pretty seamless experience. But Windows is on desktop, and time for getting comfy in the chair in front of it is at premium... So here are my attempts at alternative mobile computing. ( Yes, I could get a decent laptop and be done, but where is the fun in it? :-) )

First attempt was with Samsung Galaxy S9. Galaxy S9 is a phone with decent hardware, plus this was Samsung second (?) attempt at creating combined mobile + desktop experience for users with it's DeX environment. Like all things mobile, DeX is almost there, almost...


Step A: find a DeX Pad on Kijiji; figure that it does not allow docking the phone with the cover, get the USB C extender. All connected, works, TV Shows extended desktop, fan is whirrling - but it's sooo clunky. How am I going to use this in the car?


Lookie there - it has a fan to cool off your phone! And regular USB-A ports for mouse!

Step B: give away the Dex Pad.

Step C: Get a portable USB-C monitor for about $150. Turns out if you connect power adapter to the monitor and then connect monitor to Galaxy S9, as long as the whole thing is powered phone will turn on the DeX mode for you. Hooray, no special DeX device needed! Now I have 15'' screen, phone can be used as a mouse and bloetooth keyboard connect for typing. Top it off with a $30 power inverter for you car's cigarette lighter (who used it as a cigarette lighter in last 30 years?) and you got your mobile station!


Lot's of cables to connect before you start coding

Best of all, no jailbreak is needed to turn your phone into full development environment. Get Termux, install anything you want - it's a full Linux environment. Build, run web server locally, open it in chrome. You can even do dev console approximation in mobile Chrome using Eruda console. But... after years developing on desktop you sorely miss one single piece of software: a decent editor that can open your files in tabs and show your project tree on the left with some syntax highlighting.


Yes, this is how Termux looks on a phone - isn't it cool?

One challenge with Termux is that the its file system is not directly accessible, your editor has to have special implementation to see the files. So many editors from Google Play do not even see the project. On the other side I got recently hooked up on VS Code, which gives you same experience everywhere with pretty much all the features you need for development (still to figure out how debugging works).

It is possible to run VS Code on Galaxy S9 in Termux. You have to install few hundred node.js dependencies and kick it off. It opens, you can even edit code there without much lag. Except... once you switch to your browser to see changes, Android kills and restarts the VS Code process. VS Code is eating too much memory and Android cannot tolerate that - it has to forcibly free up memory (at least this is my present understanding), and there is no known workaround without jailbreak. I was so close to having single computing device!


Source: dev.to

At least I can run RDP on this setup and connect to my workstation when at home...
